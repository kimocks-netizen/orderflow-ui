## Architecture Decisions

### Data flow:
  Server data (orders, dashboard) → TanStack Query (features/*/queries.ts)
  Client UI state (theme, auth, sidebar) → Zustand (stores/)

### API communication:
  lib/api-client.ts → FastAPI backend (http://EC2_IP:8000/api)
  Features call api-client via features/*/api.ts
  Pages consume data via features/*/queries.ts hooks

### Forms:
  React Hook Form + Zod schemas
  Used for: login, create order
  shadcn/ui Form components for consistent field rendering

### Auth:
  JWT token stored via Zustand (persisted to cookie)
  ProtectedRoute component wraps private pages
  ApiClient auto-injects Bearer token on all requests
  401 response → clear token → redirect to /login

### Layout:
  AppLayout uses sticky Navbar (top nav) instead of sidebar — fewer nav items don't warrant a sidebar
  Background images (light/dark variants) fixed behind content via CSS
  Mask.tsx canvas particle animation layered above background
  Footer included in AppLayout for all authenticated pages
  LoginPage has its own full-screen background + Mask + theme toggle

### Theme:
  CSS variables (:root + .dark) with HSL values
  Tailwind darkMode: 'class'
  Zustand store toggles class on <html>, persists to localStorage

### Feature module pattern:
  features/<domain>/api.ts — typed fetch functions
  features/<domain>/queries.ts — TanStack useQuery hooks
  features/<domain>/mutations.ts — TanStack useMutation hooks
  Pages import from features/ — never call lib/api-client.ts directly
